# SkillHub

七牛云 SkillHub 是一个面向 AI Agent / Coding Agent 的公开 skill 仓库，用来集中维护七牛云官方可对外发布的 skill，也欢迎社区贡献通用或与七牛云生态相关的 skill。

## 项目描述

一个收录七牛云官方 skill 与社区贡献 skill 的开放仓库，面向 AI Agent 的能力扩展、工作流复用与最佳实践沉淀。

## 项目定位

- 收录七牛云官方可以公开发布的 skill
- 接收社区贡献的高质量 skill
- 为 Agent 提供可复用、可组合、可维护的能力模块
- 沉淀七牛云相关场景的最佳实践与标准接入方式

## 仓库适合什么内容

- 七牛云产品能力相关的 skill
- 面向开发者、运维、内容生产、自动化流程等场景的 skill
- 能明显提升 Agent 执行效果的领域知识、流程模板或操作规范
- 可被其他项目直接复用的通用 skill

## Skill 组织方式

每个 skill 建议使用独立目录维护，至少包含：

- `SKILL.md`：skill 的核心定义、触发条件、执行步骤与约束

建议一个 skill 对应一个目录，例如：

```text
example-skill/
└── SKILL.md
```

如果后续需要补充参考材料，也可以在 skill 目录下增加：

- `assets/`：静态资源
- `references/`：参考文档
- `scripts/`：辅助脚本

## 编写原则

- 目标明确：skill 解决的问题要清晰，避免过宽或过泛
- 触发准确：描述清楚在什么场景下触发
- 可执行：步骤应具体，避免只有概念没有操作
- 少发明结构：优先复用目标项目已有模式
- 可维护：命名清晰，约束明确，尽量减少歧义

## 贡献方式

欢迎提交 issue 或 PR 贡献新的 skill，或改进已有 skill。

建议贡献遵循以下要求：

- skill 有明确的使用场景和目标用户
- 文档完整，至少说明用途、触发条件、执行步骤和边界
- 不包含敏感信息、私有凭证或不可公开传播的内容
- 尽量保持通用性，避免强绑定个人环境
- 与仓库已有 skill 的目录结构和表达方式保持一致

## 建议的提交流程

1. 新建一个独立的 skill 目录
2. 编写或补充 `SKILL.md`
3. 自查命名、触发词和执行步骤是否清晰
4. 提交 PR，说明 skill 的用途、适用场景和预期效果

## 适用对象

- 希望为七牛云生态补充能力的外部贡献者
- 需要复用 skill 构建 Agent 工作流的开发者

## 目标

SkillHub 希望逐步成为：

- 七牛云官方 skill 的统一公开出口
- 社区共建的高质量 skill 集合
- 面向 AI Agent 能力扩展的实践样板库
